# Учёт прихода по QR — сайт на Vercel

Веб-приложение: администратор показывает QR-код на входе (обновляется каждую минуту),
сотрудники входят по ИИН и паролю и отмечают приход/уход сканированием. Считаются
опоздания и отработанные часы, фиксируются устройство и IP. Отчёт за период
выгружается в CSV и PDF.

## Архитектура
- **Фронтенд** — статические `index.html` + `app.js` (камера, генерация и сканирование QR, PDF).
- **Бэкенд** — серверные функции в папке `api/` (Node, ESM).
- **Хранилище** — Vercel KV / Upstash Redis (через REST API).
- **Безопасность** — пароли хешируются на сервере (scrypt), сессия в подписанной
  httpOnly-cookie, секрет QR-кода и проверки — только на сервере, **IP определяется сервером**.

## Развёртывание (по шагам)

1. **Создайте репозиторий** и положите туда содержимое этой папки
   (или загрузите папку напрямую в Vercel).

2. **Импортируйте проект в Vercel** → New Project → выберите репозиторий.
   Framework Preset: **Other**. Build Command оставьте пустым, Output — корень.

3. **Подключите хранилище.** В проекте Vercel → вкладка **Storage** →
   добавьте **Upstash Redis** (или Vercel KV). Vercel автоматически добавит в проект
   переменные `KV_REST_API_URL` и `KV_REST_API_TOKEN`.

4. **Добавьте переменные окружения** (Project → Settings → Environment Variables):

   | Переменная         | Назначение                                  | Пример              |
   |--------------------|---------------------------------------------|---------------------|
   | `SESSION_SECRET`   | секрет подписи сессий (длинная случайная строка) | `сгенерируйте 32+ симв.` |
   | `ADMIN_IIN`        | ИИН первого администратора (12 цифр)        | `850101300123`      |
   | `ADMIN_PASSWORD`   | пароль первого администратора               | `надёжный-пароль`   |
   | `ADMIN_FIO`        | (необязательно) имя администратора          | `Администратор`     |
   | `OFFICE_TZ`        | (необязательно) часовой пояс офиса          | `Asia/Almaty`       |

   `KV_REST_API_URL` и `KV_REST_API_TOKEN` появятся сами после шага 3.

5. **Deploy.** После деплоя откройте сайт и войдите под `ADMIN_IIN` / `ADMIN_PASSWORD`.
   Эти данные нигде не показываются на странице входа — администратор «не виден».

6. В разделе **Сотрудники** задайте начало смены и часовой пояс, заведите список
   сотрудников (ИИН, ФИО, пароль). Сотрудники входят со своих телефонов и отмечаются.

## Первый администратор создаётся один раз
Админ из переменных окружения создаётся автоматически только если хранилище пустое
(при самом первом входе). Дальше учётные записи управляются в разделе «Сотрудники».
Чтобы сбросить всё с нуля — очистите ключи `employees`, `attendance`, `settings`
в хранилище Upstash/KV.

## Локальный запуск (необязательно)
```
npm i -g vercel
vercel dev
```
Понадобятся те же переменные окружения (создайте `.env.local` или задайте через `vercel env`).

## Заметки по безопасности
- Это рабочая основа, а не аудированная система. Перед реальной эксплуатацией смените
  все секреты, ограничьте доступ и при необходимости добавьте журналирование/2FA.
- Привязка устройства основана на «отпечатке» браузера (ОС, модель где доступна,
  браузер, экран) — он не равен паспортной модели телефона; точную модель отдаёт только
  нативное приложение.
